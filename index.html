Some text to be repeatedly loaded

Some text to be luckily loaded

Matrix style loading

(Syntax)
(Hide Syntax)

<~~~ class="loading decoration"
[mode={"normal" | "luck"}]
[delay=Number]
[repeat=Number]
[event=RestartEvent]
[start=Anything]
[interval=Number]
[direction={"ltr" | "rtl"}]
[shuffle={"rest" | "all" | "alphabet"}]
[delimiter=String]
[skip=Number]
[blink=Number]
[whenDone=JSCodeString]
>
content text
</~~~>


Details:
mode :The mode of the loading decoration
  • "normal": each step a new letter is "loaded"
  • "luck":     a new letter is "loaded" only when the viewed position is with the letter it should
delay :The miliseconds the letter loading is delayed. This ensures n ms of shuffling content
repeat :The number of times the effect is repeated
  • 0: repeat till infinity
event :The event at which the effect should start/restart
start :If any value is given the effect will start immediately
interval :The interval between steps in miliseconds
  • default: 150
direction :The direction from which the letters are being "loaded"
  • "ltr": Left-to-Right (default)
  • "rtl": Right-to-Left
shuffle :Specifies what to shuffle among
  • "rest":         The shuffled letters are only of the unloaded ones
  • "all":           The shuffled letters are among all - loaded and unloaded
  • "alphabet":  The shuffled letters are the letters from the alphabet. Note, no uppercase included and length might be less (never more) than it should
delimiter :A delimiter string/html to be put between each letter
skip :The number of steps that are skipped / no letter is loaded. In other words, the steps it takes for a single letter to load
blink :Every N steps blink. For a single step display a fully shuffled text
whenDone :Javascript code string to be executed when effect is done (time to repeat)
  • this refers to the node.decoration object

Matrix style pour

(Syntax)
(Hide Syntax)

<~~~ class="pour decoration"
[direction={"ltr" | "rtl" | "cts" | "random"}]
[tags={"instant" | "delayed"}]
[event=RestartEvent]
[start=Anything]
[interval=Number]
[delimiter=String]
[blink=Number]
[rewind=Number]
[wait=Number
[back=Number] ]
[pad=String]
>
content HTML
</~~~>


Details:
direction :The direction in which the text is poured
  • "ltr": Left-to-Right
  • "rtl": Right-to-Left
  • "cts": Center-to-Sides. Instant tags don't work here (yet)
  • "random": Each step a random letter is loaded. Instant tags don't work here (yet)
tags :If tags should be poured instantly or a tag letter per step
  • "instant": The tags are poured as if they were one letter
  • "delayed": The tags are poured like the rest of the text, though are not displayed after the "<"
event :The event at which the effect should start/restart
start :If any value is given the effect will start immediately
interval :The interval between steps in miliseconds
  • default: 150
delimiter :A delimiter string/html to be put between each letter
  • "ltr": Left-to-Right (default)
  • "rtl": Right-to-Left
blink :In N steps have one blank
rewind :In N steps rewind / start over. N stacks so after the first rewind it will rewind in N*2, then in N*3 ... turns
wait :In N steps wait / don't progress
back :Used with "wait"! In "wait" steps go back N steps
pad :Used only when "direction"=="random"! Each hidden / unpoured letter is replaced by this string / HTML

(Syntax)

<~~~ class="radio decoration"
[group=IdentifierString]
[event=EventString]
[property=String]
[value="ValueShown,ValueHidden"]
[start=Anything]
[decorate=Anything]
[exclude={"me" | "this"}]
[whenOpened=JSCodeString]
>
content HTML
</~~~>


Details:
group :The radio group which's elements to toggle
  • group="identifier1,identifier2,identifier3..."
  • "#ID": find an element by id ID
  • ".CLASS1.CLASS2": find elements by CLASS1 and CLASS2 (order NOT required)
  • ".CLASS1 CLASS2": find elements by "CLASS1 CLASS2" (order IS required)
  • "NAMED": find elements by name NAMED
event :A list of events to listen for
  • event="evn1,evn2,evn3..."
  • "ID@EVN:IND" : Find element by id ID and at EVN display index IND
  • "EVN" : Set the default event to EVN and apply it to all group targets that have no event assigned
property :The property which's value should be modified
  • default "display"
value :Value(s) which the property should take
  • value="ValueShown,ValueHidden"
  • if just one value is supplied (no comma) -> '',ValueHidden
  • default "none"
start :If any value is given the effect will start immediately
decorate :If any value is given children decorations will not be modified / paused when parent elements are toggled
exclude :If the group must not include the calling object (self). The calling object is included by default so this is necesseray
whenOpened :javascript Code String to be executed when an element is shown
